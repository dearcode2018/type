
/* 常见枚举设计 */
私有构造方法一个、属性若干、公开方法若干、

枚举实现接口-设计
实现若干个接口，方法可以整体实现，也可以让各个
实例单独实现.

枚举属性-设计
// 枚举实例的名称，可以设计为中文名、英文名
String name;

// 值，code，一般可以为英文缩写/编码格式
String value

// 描述信息，详细说明或中文性表述/中文全称
String description

// 索引，即用 0,1,2...来表示枚举实例的值
int index

数据表-存储: 数字、编码、英文名称 表示
客户端-展示: 中文名称、中文描述、value可以作为下拉控件
的value值，然后服务端可以根据该value值来获取相应的枚举实例对象
<select name="">
	<option value="" />
</select>


枚举方法-设计
枚举只能在内部进行构造，对外不提供公构造方式，
构造方法可以有针对性和属性赋值挂钩
抽象方法: 若需要一个统一方法声明但又不统一实现，
而是让各个实例去单独实现，则可以采用此方式.
 枚举的属性方法，一般只提供get，而不提供set方法，
 特殊情况例外.
 根据value来获取枚举实例，value此时设计为String类型
public static T getInstance(final String value);
